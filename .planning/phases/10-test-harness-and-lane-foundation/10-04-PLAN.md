<interfaces>
Current implicit payload protocol (to replace):

```text
scripts/scheduled_consolidate.py -> print(json.dumps(result))    # last stdout line
scripts/governance_worker.py     -> print(json.dumps(result))    # last stdout line
tests/support/runtime_drivers.py -> last JSON-parseable dict line wins
```

Target payload protocol:

```text
scripts -> print("===AUTOMEM_PAYLOAD_BEGIN===")
           print(json.dumps(result, ensure_ascii=False))
           print("===AUTOMEM_PAYLOAD_END===")

runtime_drivers -> read the single block between the sentinels; payload=None if absent;
                   payload may be any JSON value (dict/list/scalar), with dict-ness
                   enforced only where callers explicitly require it.
```

FakeMemory structured-content contract (new):

```python
messages = [{"role": "user", "content": [{"type": "text", "text": "alpha"}]}]
# expected record["memory"] == "alpha"   (not Python repr of the list)
```

FakeMemory aliasing contract (new):

```python
rec = memory.get("mem_1")
rec["metadata"]["status"] = "mutated"
# memory.get("mem_1")["metadata"].get("status") must NOT be "mutated"
```

Adversarial findings being closed here (ten items from 2026-04-17 review):

1. `_parse_payload` reverse-scan is fragile to log lines / pretty-printed JSON / non-dict JSON (Task 1).
2. `_extract_text` returns Python repr for list-of-parts content (Task 2).
3. Records are aliased, get/get_all/search leak internal dict references (Task 3).
4. `**params` silently accepts unknown kwargs and typoed keys (Task 3).
5. `created_at` and `score` hardcoded, making time/ranking tests meaningless (Task 3).
6. `_new_lock_path` uses fresh UUID per call, defeating single-flight tests (Task 4).
7. `extra_env.update(extra_env)` allows callers to silently override helper contract keys (Task 4).
8. `os.environ.copy()` passes every developer env var into subprocesses (Task 4).
9. `uv run` cold start per slow-lane test (deferred to Phase 13 canary lane policy).
10. No contract-parity test between FakeMemory and a real mem0-backed implementation (deferred to v2 ADV-01).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Runtime script stdout -> runtime driver payload | An unstructured "last JSON line wins" rule lets any future log line silently replace the business payload |
| Developer environment -> runtime subprocess | `os.environ.copy()` leaks arbitrary developer env vars into child processes used by slow-lane tests |
| Test code -> FakeMemory internal state | Shared dict references let test code mutate the store through returned objects; `**params` absorbs typoed kwargs |
| Helper contract -> `extra_env` | Callers can overwrite helper-guaranteed keys (mode, attempts, lock, idempotency) without any guard |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-04-01 | T | `tests/support/runtime_drivers.py` `_parse_payload` | mitigate | Replace reverse-scan with sentinel-delimited payload extraction so business payload is not spoofable by ordinary log lines. |
| T-10-04-02 | T | `tests/support/fake_memory.py` `_extract_text` | mitigate | Treat list-of-parts content as structured data; raise on shapes the helper does not understand instead of coercing to a Python repr string. |
| T-10-04-03 | T | `tests/support/fake_memory.py` record accessors | mitigate | Return defensive copies from `get`/`get_all`/`search`/`add` so tests cannot corrupt store state via aliasing. |
| T-10-04-04 | T | `tests/support/fake_memory.py` `**params` | mitigate | Accept only the kwargs the production backend uses and raise `TypeError` on typoed keys. |
| T-10-04-05 | R | `tests/support/runtime_drivers.py` lock/idempotency defaults | mitigate | Stable default lock path and idempotency key per helper; random values become explicit opt-in so single-flight and replay tests exercise real behavior. |
| T-10-04-06 | E | `tests/support/runtime_drivers.py` `extra_env` | mitigate | Refuse silent overrides of contract keys; require `allow_contract_override=True` when a caller knowingly wants to replace them. |
| T-10-04-07 | I | `tests/support/runtime_drivers.py` env construction | mitigate | Start child env from a module-level allowlist instead of the caller's whole environment, to avoid leaking developer env vars into subprocesses. |
</threat_model>
