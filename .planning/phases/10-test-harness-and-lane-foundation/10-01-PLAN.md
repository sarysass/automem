<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| test harness -> backend process | Untrusted test inputs cross into the real FastAPI app and must still pass normal auth and routing |
| harness env -> temp SQLite and lock files | Mis-scoped paths can leak state across tests or touch repo-default files |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01-01 | T/I | `tests/support/live_backend.py` | mitigate | Build env from a per-test temp directory, set explicit temp paths for `TASK_DB_PATH`, `HISTORY_DB_PATH`, `AUTOMEM_WORKER_LOCK_FILE`, and `MEMORY_CONSOLIDATE_LOCK_FILE`, and assert temp-path values in `tests/test_harness_foundation_live.py`. |
| T-10-01-02 | E | `backend/main.py` auth path under test | mitigate | Probe `/healthz` only through normal `X-API-Key` auth using `ADMIN_API_KEY=test-admin`; do not depend on `PYTEST_CURRENT_TEST` or insecure startup bypasses. |
| T-10-01-03 | S | `tests/support/fake_memory.py` bootstrap | mitigate | Inject `FakeMemory` inside the dynamic backend import path before requests are served so the harness cannot silently fall back to real mem0/Qdrant dependencies. |
</threat_model>
